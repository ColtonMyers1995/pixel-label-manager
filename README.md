# Pixel Label Manager (ArkOS) by ColtonMyers1995
-Custom-Collection Label-Maker-

A controller-friendly tool for creating custom system/category labels for the PIXEL OS theme.
The tool operates within ArkOS, giving users a full label-making toolkit.
The created labels can be applied directly to custom-collections per view (icons, logos, systems)

Additional: PIXEL OS's unique design allows this tool to create up to 3 custom images per target name, 1 for each view (icons, logos, systems)
            (Creating a logo/icon/system with the same name will allow the user to see each created image/label while using the respective view.)

## Features
- Create labels, icons and systems
- Multiple logo types
- Access massive icon library (default + custom packs)
- Grid-based text input (controller-based)
- Font library
- Color library
- Automatic PNG generation end exporting

## Installation
Copy the script to:
```
/roms/tools/
```

Make executable if needed (not needed for ArkOS 2.0):
```
chmod +x Custom-Collection Label-Maker.sh
```

## Usage
- Launch from Tools in ArkOS
- Design your logo/icon (choose style, color, sizes, text, image, padding, spacing, type (logo, icon, system))

## Additional Info
- The 3 label types are paired to the view types in PIXEL OS; to see created icons, set to icons, etc
- Logos are more complex than icons/systems, offering MANY customization options and multiple styles
- Logos are automatically pushed to icon and system views if no custom icon or system PNG exists
  (create a separate 'icon' and/or 'system' PNG(s) to avoid created logos appearing in icon/system views)

## Simple Walkthrough of tool
0. Install tool
1. Open tool from tools menu
2. Create logo/icon/system image
3. Create custom-collection with same name as label/image you just created
4. Enable the custom collection
5. Turn off "group unthemed systems"
6. Restart ES with quit menu or toolkit
7. Find your custom-collection in system list
   (make sure you are using the same view as the label you made (logo/icon/image) in theme settings)
8. Label is applied and visible
9. [OPTIONAL] Use the tool to recreate the image again if you want to change parts of the label you created.
   (recreating a label after step 8 does not require restarting ES or remaking a custom-collection to apply)
